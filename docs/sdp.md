## 2. Project Overview (Обзор продукта)

### 2.1 Project Purpose, Scope, and Objectives (Назначение, цели и контекст продукта)
*[Кратко опишите назначение проекта, цели его разработки, какие артефакты будут получены в результате работы над ним]*

Проект решает ряд задач:
* Оптимизирует управление правохранительными органами путем автоматизации.
* Решает проблему этичной, непредвзятой проверки граждан на способность совершать преступления.  
Как следствие, соотношение цена/качество для правохранительных органах улучшается. Проект экономит средства/улучшает благосостояние в регионе, где он развернут.  

Цель разработки - извлечение прибыли путём предоставления продукта по подписке.

Артефакты, полученные в резульате разработки см. в п. 2.3

*[Перечислите ограничения, накладываемые на данный план разработки проекта - бюджет, персонал, сроки и т.д.]*

* Бюджет проекта - 6 ящиков пива. 

### 2.3 Project Deliverables (Ожидаемые результаты проекта)
*[Перечислите артефакты, который будут созданы в результате проекта.]*

По окончании разработки будут получены следующие сервисы c соответствующим функционалом, из которых состоит продукт:

- Civilla - Video - сбор информации о психопаспортах.
- Civilla - Database - хранение информации о психопаспортах.
- Civilla - Analysis - обработка информации, вынесение решения о допустимой мере пресечения преступлений.
- Civilla - Maps - сервис для аналитиков, наглядно отображающий на карте концентрацию высоких коэфициентов преступности.
- Civilla - Police Control - сервис авторизации и оповещения полиции.
- Civilla - Dominator Service - сервис разрешения доступа к применинию оружия (в соответствии с коэф. преступности).

Помимо сервисов, будут получены инструменты для развертывания продукта, его тестирования и мониторинга его работоспособности.

### 3.3 Roles and Responsibilities (Роли и обязанности)
*[Приведите описание ролей в проекте и перечислите их обязанности.]*

* Project Manager 1 (interaction with customer, business development planning) - Координирует взаимодействие с заказчиком, доносит требования заказчика до разработчиков, фокусирует команду на бизнес-целях.

* Project Manager 2 (technical development planning, routine management) - Устанавливает практики разработки, планирует разработку, исходя из целей бизнеса, контролирует качество проекта.  

* Deployment Manager - планирует, как поставлять продукт заказчику.

* Configuration Manager - Менеджер конфигурации предоставляет всю инфраструктуру и среду управления конфигурациями для команды разработчиков продукта. Поддерживает деятельность по разработке продукта, таким образом разработчики и интеграторы имеют соответствующие рабочие пространства для построения и тестирования своей работы.

* Change Control Manager - Контролирует процесс внесения изменений в проект.  

* Test Manager - Определяет практики тестирования, обеспечивает успешность тестирования продукта.

* Requirements Reviewer 1 (Business review) - Контролирует требования к продукту с точки зрения бизнеса. 

* Requirements Reviewer 2 (technical review) - Контролирует требования к продукту с технической точки зрения.

* Architecture Reviewer - Контролирует архитектуру программного обеспечения в целом.

* Test Analyst - Отвечает за определение случаев, которые необходимо протестировать.

* User Interface Designer - Координирует разработку пользовательского интерфейса.

* Designer - Разрабатывает компоненты продукта, определяет внутренние взаимосвязи подсистем.

* Implementer - Реализует и тестирует компоненты продукта.

* Code Reviewer - Контролирует качество кода, проводит code review.

* Test Designer - Определяет подход к тестирования определенной возможности продукта.

* Tester - Проводит тестирование, анализирует результаты тестирования.
